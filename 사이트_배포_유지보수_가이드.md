# 포트폴리오 사이트 배포 & 유지보수 가이드 (업데이트)

이제 사이트는 `index.html` + `main.js` 두 파일로 구성돼요 (토글 인터랙션 때문에 JS가 분리됨). 매번 GitHub 웹사이트에서 파일 업로드하는 게 귀찮으시다면, **GitHub Desktop**을 쓰는 걸 강력 추천드려요. 아래 방법대로 한 번만 설정해두면, 그 다음부터는 로컬 파일을 수정하고 버튼 하나만 누르면 자동으로 사이트에 반영됩니다.

---

## 1. GitHub Desktop으로 "귀찮음 없는" 자동 동기화 설정하기

### 1) GitHub Desktop 설치
https://desktop.github.com 에서 다운로드 → 설치 → 본인 GitHub 계정으로 로그인

### 2) 기존 저장소를 컴퓨터로 "클론"하기
1. GitHub Desktop 실행 → `File` → `Clone Repository`
2. 목록에서 `JADEB0B.github.io` (또는 본인 저장소명) 선택
3. 저장할 폴더 지정 후 `Clone`
4. 이제 그 폴더가 내 컴퓨터의 실제 폴더이면서 동시에 GitHub 저장소와 연결된 상태가 됩니다

### 3) 파일 수정은 이 클론된 폴더에서
앞으로 `index.html`, `main.js`를 고칠 때는 **이 클론된 폴더 안의 파일을 직접 열어서 수정**하세요. (메모장, VS Code, 아무 편집기나 가능)

### 4) 수정 후 반영 (매번 이 3단계만 반복)
1. GitHub Desktop을 열면 변경된 파일이 자동으로 감지되어 보여요
2. 하단 `Summary`에 간단히 뭘 고쳤는지 한 줄 적고 `Commit to main` 클릭
3. 상단 `Push origin` 버튼 클릭

이게 끝입니다. 웹사이트에 로그인해서 파일을 드래그할 필요 없이, 로컬에서 고치고 버튼 두 번 누르면 1분 안에 실제 사이트(`https://jadeb0b.github.io`)에 반영돼요.

## 2. 로컬에서 미리보기 (중요: 파일이 2개가 되면서 방법이 달라졌어요)

파일이 `index.html` 하나였을 때는 그냥 더블클릭해서 열어도 됐지만, 지금은 `main.js`를 따로 불러오고 보안 정책(CSP)도 있어서 **더블클릭으로 열면 토글 애니메이션이 안 될 수 있어요.**

대신 터미널(맥은 `터미널` 앱)에서 아래처럼 간단한 로컬 서버를 띄우고 확인하세요.

```bash
cd 클론한_폴더_경로
python3 -m http.server 8000
```

그다음 브라우저에서 `http://localhost:8000` 접속하면 실제 배포 환경과 거의 동일하게 미리 볼 수 있습니다. (`Ctrl + C`로 서버 종료)

## 3. 지금 구조 이해하기

사이트는 우측 상단 토글 버튼으로 두 가지 모드가 전환돼요.

- **Public** (`<div id="publicView">`): About / Education / Project / Writing / 3D Model / Contact — 이력서 성격의 공개용
- **Personal** (`<div id="privateView">`): Playlist / Favorite Anime / 지하철 — 취향 공유용

각 섹션은 `<section>` 또는 `<div class="glass-card">` 블록으로 나뉘어 있어서, 원하는 블록을 찾아 그 안의 텍스트만 바꾸면 안전합니다.

### 아직 비어 있는 자리 (준비되면 알려주시면 채워드릴게요)
- `#writing` 섹션: 글쓰기 보고서 텍스트
- `#model3d` 섹션: Fusion360 STL/GLB 파일 → 웹 3D 뷰어
- `.anime-grid`의 `placeholder` 카드 4개: 실제 애니 제목 + 이미지

---

## 4. 자주 하는 수정

### 텍스트 바꾸기
바꾸고 싶은 문장을 찾아 그대로 다른 문장으로 교체하면 됩니다. `<p>`와 `</p>` 사이만 바꾸면 안전해요.

### 링크 바꾸기
`href="..."` 부분만 원하는 주소로 교체하면 됩니다.

### 색상 바꾸기
`<style>` 안 `:root { ... }`의 색상 값들을 바꾸면 전체 톤이 바뀝니다. Public 쪽은 흑백 톤, Personal 쪽은 `.glass-blobs`의 파스텔 색상들입니다.

## 5. 나중에 여유가 되면

- **커스텀 도메인**: GitHub Pages Settings → Custom domain에서 연결
- **애니 카드에 실제 이미지 넣기**: `images/` 폴더 만들어 표지 이미지 업로드 후 `.anime-tile`에 `<img>` 추가
- **3D 모델**: Fusion360에서 `File → Export → STL` 또는 `GLB`로 내보낸 뒤 저장소에 업로드하면, `<model-viewer>` 컴포넌트로 웹에서 회전/확대 가능하게 연결 가능
